# Implementation Plan: File Staging Feature

## Overview

Add `stageFile` capability to the git-quest VS Code extension. This allows users to stage modified files via a context menu command, with the tree view refreshing automatically.

## Step 1: Add `stageFile` to GitService

**File:** `src/git-service.ts`

Add a new public method after `getRepositoryPath()` (around line 30):

```ts
async stageFile(filePath: string): Promise<void> {
  if (!this.rootPath) {
    throw new Error('No workspace folder is open');
  }
  return new Promise((resolve, reject) => {
    exec(`git add "${filePath}"`, { cwd: this.rootPath! }, (error) => {
      if (error) {
        reject(new Error(`Failed to stage file: ${error.message}`));
        return;
      }
      resolve();
    });
  });
}
```

**Key decisions:**
- Uses `git add "<filePath>"` with quotes to handle paths with spaces
- Throws on failure so the command handler can show an error message
- Follows the same `exec` pattern as existing methods

## Step 2: Add `contextValue` to Tree Items

**File:** `src/tree-view.ts`

Modify the tree item creation for modified files (lines 54-60) to include `contextValue`:

```ts
modifiedNode.children = this.cachedInfo.modifiedFiles.map((filePath) => {
  const item = new GitQuestTreeItem(filePath, vscode.TreeItemCollapsibleState.None, {
    command: 'vscode.open',
    title: 'Open File',
    arguments: [vscode.Uri.file(path.join(this.cachedInfo!.rootPath!, filePath))],
  });
  item.contextValue = 'modifiedFile';
  return item;
});
```

Similarly for staged files (lines 68-74), add `contextValue = 'stagedFile'`.

**Why:** The `contextValue` is used in the `when` clause of the menu contribution to target only modified files.

## Step 3: Register the Command

**File:** `src/extension.ts`

Add a new command registration after the existing commands (around line 31):

```ts
context.subscriptions.push(
  vscode.commands.registerCommand('git-quest.stageFile', async (fileItem: GitQuestTreeItem) => {
    const filePath = fileItem.label;
    try {
      await gitService.stageFile(filePath);
      treeDataProvider.refresh();
    } catch (error) {
      vscode.window.showErrorMessage(`Failed to stage file: ${error}`);
    }
  })
);
```

**Note:** Need to import `GitQuestTreeItem` from `./tree-view`.

**Key decisions:**
- The command receives the tree item, extracts `label` (which is the file path)
- Calls `gitService.stageFile()` for the git operation
- Calls `treeDataProvider.refresh()` to update the UI
- Shows error message on failure

## Step 4: Update package.json

**File:** `package.json`

### 4a. Add command declaration

Add to the `commands` array:

```json
{ "command": "git-quest.stageFile", "title": "Stage File" }
```

### 4b. Add menu contribution

Add a new `menus` section to `contributes`:

```json
"menus": {
  "view/item/context": [
    {
      "command": "git-quest.stageFile",
      "when": "view == git-quest-view && viewItem == modifiedFile",
      "group": "inline"
    },
    {
      "command": "git-quest.stageFile",
      "when": "view == git-quest-view && viewItem == modifiedFile",
      "group": "1_modification"
    }
  ]
}
```

**Key decisions:**
- `inline: true` shows the button directly in the tree item row
- The `1_modification` group places it in the context menu
- `when` clause ensures it only appears on modified files in our tree view
