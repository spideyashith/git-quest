### 1. `src/git-service.ts` — Add `unstageFile`

Add a new method after `stageFile` (line 45), following the same pattern:

```typescript
async unstageFile(filePath: string): Promise<void> {
  if (!this.rootPath) {
    throw new Error('No workspace folder is open');
  }
  return new Promise((resolve, reject) => {
    execFile('git', ['reset', 'HEAD', filePath], { cwd: this.rootPath! }, (error) => {
      if (error) {
        reject(new Error(`Failed to unstage file: ${error.message}`));
        return;
      }
      resolve();
    });
  });
}
```

Uses `git reset HEAD <file>` which is the standard way to unstage a file while keeping working directory changes.

  vscode.commands.registerCommand('git-quest.unstageFile', async () => {
    const item = treeView.selection[0];
    if (!item) {
      vscode.window.showWarningMessage('No file selected to unstage.');
      return;

    const fileItem = item as GitQuestTreeItem;

      vscode.window.showErrorMessage(`Failed to unstage file: ${error}`);
    }
  })

```

## Design for Commit Feature

The implementation will follow the same clean‑architecture pattern used for staging/unstaging:

1. **GitService** – add a `commit(message: string): Promise<void>` method that runs `git commit -m <message>` inside the repository root. It will:
   - Throw if no workspace is open.
   - Return a promise that resolves on success or rejects with a descriptive error.

2. **Extension.ts** – register a new command `git-quest.commit`.
   - Prompt the user for a commit message with `vscode.window.showInputBox`.
   - If the user cancels or enters an empty/whitespace‑only string, show an error (`vscode.window.showErrorMessage('Commit message cannot be empty')`).
   - Call `gitService.commit(message)`.
   - On success, refresh the tree view (`treeDataProvider.refresh()`) and show a success notification (`vscode.window.showInformationMessage('Commit succeeded')`).
   - On failure, show the error message.

3. **package.json** – add the command definition to `contributes.commands`. No context‑menu entry is required (the command can be invoked from the command palette).

4. **TreeDataProvider** – no changes needed beyond the existing `refresh` method; the UI will update after the commit.

5. **Validation** – the command enforces a non‑empty commit message before invoking Git.

async getRecentCommits(limit: number = 10): Promise<GitCommit[]> {
  if (!this.rootPath) {
    throw new Error('No workspace folder is open');
  }
  return new Promise((resolve, reject) => {
    // Format: short_hash|message|author|date
    exec('git log --max-count=' + limit + ' --pretty=format:"%h|%s|%an|%ad"', { cwd: this.rootPath! }, (error, stdout) => {
      if (error) {
        reject(new Error(`Failed to get commit history: ${error.message}`));
        return;
      }
      const commits: GitCommit[] = [];
      const lines = stdout.split('\n').filter((line) => line.length > 0);
      for (const line of lines) {
        const [hash, message, author, date] = line.split('|');
        if (hash && message && author && date) {
          commits.push({ hash, message, author, date });
        }
      }
      resolve(commits);
    });
  });
}
```

## Git Log Format Explanation

- `%h` = abbreviated commit hash (short)
- `%s` = commit subject (message)
- `%an` = author name
- `%ad` = author date (respects `--date` format, defaults to ISO-like format)

Using `|` as delimiter since commit messages can contain spaces but not pipes.
