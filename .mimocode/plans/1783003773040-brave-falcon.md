# Plan: Git Quest — Add GitService + Dynamic Tree View

## Goal
Add a GitService class that detects workspace, checks for git repo, and gets current branch. Update the tree view to display dynamic data from GitService.

---

## Files to Change (3 total)

| File | Action |
|---|---|
| `src/git-service.ts` | **Create** — all Git logic |
| `src/tree-view.ts` | **Modify** — dynamic tree based on Git state |
| `src/extension.ts` | **Modify** — wire up GitService |

---

## Step 1: Create `src/git-service.ts`

### Interface
```typescript
export interface GitRepositoryInfo {
  isGitRepository: boolean;
  rootPath: string | null;
  currentBranch: string | null;
  error?: string;
}
```

### GitService Class
- `constructor()` — detects workspace via `vscode.workspace.workspaceFolders`
- `getRepositoryPath(): string | null` — returns first workspace folder path
- `isGitRepository(): Promise<boolean>` — checks for `.git` directory using `fs.promises.stat`
- `getCurrentBranch(): Promise<string | null>` — executes `git branch --show-current` via `child_process.exec`
- `getRepositoryInfo(): Promise<GitRepositoryInfo>` — combines all above into structured response

### Error handling
- Returns null/empty states instead of throwing
- Async operations wrapped in try/catch
- All git commands run in the workspace root directory

## Step 2: Update `src/tree-view.ts`

### Changes
- Import `GitService` and `GitRepositoryInfo`
- `GitQuestTreeDataProvider` constructor takes `GitService` as parameter
- `getChildren()` becomes async — calls `gitService.getRepositoryInfo()`
- When `isGitRepository` is false → return item with "No Git repository found"
- When `isGitRepository` is true → return item with "Current Branch: {branch}"
- Keep the existing 3 placeholder items (Repository, Current Quest, Achievements) below the git status
- `refresh()` clears cache and fires `onDidChangeTreeData` event

### Tree items layout (when git repo found)
1. **Current Branch: main** (or whatever branch)
2. ---
3. Repository
4. Current Quest
5. Achievements

### Tree items layout (when NOT git repo)
1. **No Git repository found**
2. ---
3. Repository
4. Current Quest
5. Achievements

## Step 3: Update `src/extension.ts`

### Changes
- Import `GitService` from `./git-service`
- Create `GitService` instance before `GitQuestTreeDataProvider`
- Pass `GitService` to `GitQuestTreeDataProvider` constructor
- Keep existing tree view registration and placeholder commands

### Wiring flow
```typescript
const gitService = new GitService();
const treeDataProvider = new GitQuestTreeDataProvider(gitService);
const treeView = vscode.window.createTreeView('git-quest-view', { treeDataProvider });
```

---

## Architecture

```
extension.ts
  ├── creates GitService (workspace detection + git commands)
  ├── passes GitService to GitQuestTreeDataProvider
  └── registers tree view + commands

git-service.ts
  ├── detects workspace folders
  ├── checks for .git directory
  ├── executes git commands
  └── returns GitRepositoryInfo interface

tree-view.ts
  ├── receives GitService via constructor
  ├── calls getRepositoryInfo() in getChildren()
  └── displays dynamic data based on git state
```
